# WHY.md — なぜsiegeNginを作ったのか

## PRI:130型認知との相性

視覚・空間的把握が突出した認知スタイル（WAIS-IV PRI:130）にとって、「見えているものを言葉にする」コストは特に高い。頭の中で4K映像が流れているのに、言語出力が480p。

siegeNginは言語化を最小限にして、視覚的な「指さし」でAIに意図を伝える。GPU脳の出力先としてのAI。

## 10年越しの城壁突破

人間のデザイナーに「こういうUI」を伝えるのに10年かかっても伝わらなかったものが、AIとの協働で10分で形になった。言語化のボトルネックを迂回した瞬間だった。

## 壁を溶かす

siegeNginの作者が一貫して取り組んできたのは「壁を溶かす」こと。

- 地域の壁 → BizenDAO（web3 × 備前焼）
- 技術の壁 → Vibe Coding
- 構造の壁 → XPathGenie
- 言語化の壁 → siegeNgin

ルーテッドコスモポリタニズム — 地域に根ざしながら世界に開く。siegeNginもまた、ローカルのAIアシスタントとグローバルなWebの間の壁を溶かすツール。
